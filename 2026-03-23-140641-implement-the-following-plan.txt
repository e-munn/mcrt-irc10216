│ Context                                                                      │
│                                                                              │
│ Reproduce the solid line of Figure 1 from Crosas & Menten (1996, ASP Conf.   │
│ Series 93, 98) -- the kinetic temperature profile T_K(r) of the              │
│ circumstellar envelope of IRC+10216. The paper solves the thermal balance    │
│ and radiative transfer self-consistently using a Monte Carlo method. The     │
│ goal is a self-contained Python implementation that outputs a log-log plot   │
│ of T_K vs radial offset matching the paper's result.                         │
│                                                                              │
│ Strategy: Two-Phase Approach                                                 │
│                                                                              │
│ Phase A (LVG/Sobolev approximation): Build the full thermal balance solver   │
│ first using the Large Velocity Gradient (escape probability) approximation   │
│ for CO line cooling. This runs in seconds, gives a qualitatively correct     │
│ temperature profile (~10-30% of the full MC), and validates all the physics  │
│ before investing in the MC engine.                                           │
│                                                                              │
│ Phase B (Full Monte Carlo): Replace the LVG radiative transfer with a Bernes │
│  (1979) Monte Carlo engine. This is computationally expensive (~hours) but   │
│ matches the paper's method exactly.                                          │
│                                                                              │
│ I recommend implementing Phase A first, verifying the temperature profile    │
│ shape, then proceeding to Phase B only if the user wants higher fidelity.    │
